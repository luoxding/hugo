49 /// Find at least 2 parallel (m and n) and 1 reduction (k) dimension candidates
50 /// that form a matmul subcomputation within `linalgOp`.
51 /// These dimensions are such that:
52 /// 1. The m dimension is involved in an outer-product along LHS
53 /// (i.e. it is a permutation on RES and LHS and does not appear in RHS).
54 /// 2. The n dimension is involved in an outer-product along RHS
55 /// (i.e. it is a permutation on RES and RHS and does not appear in LHS).
56 /// 3. The k dimension appears as a permutation on LHS and RHS.
57 /// 4. m, n and k appear only once in any given indexing.
58 /// 5. Optional batch dimensions that appear in all operands are captured.
59 /// This allows e.g. detecting that some contraction is embedded within
60 /// `linalgOp` with some orthogonal heuristic.
61 /// When multiple dimension occurrences exist that match `batch`, `m`, `n`, or
62 /// `k`, indices are returned in sorted order.
63 /// Returns a failure if any of `m`, `n` or `k` is empty.
64 FailureOr<ContractionDimensions> inferContractionDims(LinalgOp linalgOp);

83 /// Find at least 1 parallel (output_image) and reduction (filter_loop)
84 /// dimension candidates that form a convolution subcomputation within
85 /// `linalgOp`. The LHS is assumed to be the convolution input while the
86 /// RHS is assumed as the filter.
87 /// These dimensions are such that:
88 /// 1. Optional batch dimensions that appear in the input and filter.
89 /// 2. The output_image dimension is involved in a cross-correlation along LHS
90 /// (i.e. it is a permutation on RES and LHS and has an associated
91 /// filter_loop in RHS).
92 /// 3. Optional output_channel dimension is involved in an outer-product along
93 /// RHS (i.e. it is a permutation on RES and RHS and does not appear in
94 /// LHS).
95 /// 4. Optional input_channel dimension appears as a permutation on LHS and
96 /// RHS.
97 /// 5. The filter_loop dimension appears as a permutation on the RHS and
98 /// represents the shape of the kernel cross-correlated along a
99 /// corresponding output_image dim.
100 /// 6. The input_channel dimension appears as a permutation on LHS and RHS.
101 /// 7. All dimensions appear only once in any given indexing map.
102 /// This allows e.g. detecting that some convolution is embedded within
103 /// `linalgOp` with some orthogonal heuristic.
104 /// When multiple dimension occurrences exist that match any classification
105 /// indices are returned in sorted order.
106 /// Returns a failure if `output_image` (and implicitly `filter_loop`) is empty.
107 FailureOr<ConvolutionDimensions> inferConvolutionDims(LinalgOp linalgOp);

118 /// Returns true if the block contains a contraction of the following form:
119 ///
120 /// %0 = <elemwise>(permutation-of(cu(block-argument-0),
121 /// cu(block-argument-1)))
122 /// %1 = <reduce>(permutation-of(cu(%0), cu(block-argument-2)))
123 /// return-like cu(%1)
124 ///
125 /// where <elemwise> and <reduce> are binary operations constituting a
126 /// contraction (in the canonical case, <elemwise> is a multiplication and
127 /// <reduce> is an addition). The name and other properties of these operations
128 /// are checked by `isaPair`. All operands of all operations may be supplied
129 /// through a chain of side effect-free unary operations, such as casts, which
130 /// is denoted as `cu` above.
131 ///
132 /// When the body does not contain a contraction, a more precise description of
133 /// the failed precondition is send to the `errs` stream, if provided.
134 bool isContractionBody(Block &block,
135  function_ref<bool(Operation *, Operation *)> isaPair,
136  llvm::raw_ostream &errs = mlir::thread_safe_nulls());
